Introduction à l’instruction MySQL SELECT FROM

L'instruction SELECT permet de sélectionner des données provenant d'une ou plusieurs tables. Pour écrire une instruction SELECT en MySQL, vous utilisez cette syntaxe :

SELECT liste_de_sélection
FROM nom_table;

Explication de la syntaxe

  • Premièrement, indiquez une ou plusieurs colonnes à partir desquelles vous souhaitez sélectionner des données après le mot-clé SELECT. Si la liste contient plusieurs colonnes, séparez-les par une virgule (,).
  • Deuxièmement, indiquez le nom de la table à partir de laquelle vous voulez extraire les données après le mot-clé FROM.

Le point-virgule (;) marque la fin d’une instruction. Il est obligatoire si vous enchaînez plusieurs requêtes.

Par convention, les mots-clés SQL sont écrits en majuscules, mais SQL n’est pas sensible à la casse :

select liste_de_sélection
from nom_table;

MySQL évalue d’abord la clause FROM, puis la clause SELECT.

Exemples d’utilisation de SELECT FROM

Supposons que nous ayons une table employees avec les colonnes : employeeNumber, lastName, firstName, extension, email, officeCode, reportsTo et jobTitle.

1) Sélection d’une seule colonne

SELECT lastName
FROM employees;

2) Sélection de plusieurs colonnes

SELECT 
    lastName, 
    firstName, 
    jobTitle
FROM employees;

3) Sélection de toutes les colonnes

Exemple avec les noms de colonnes :

SELECT employeeNumber,
       lastName,
       firstName,
       extension,
       email,
       officeCode,
       reportsTo,
       jobTitle
FROM employees;

Ou avec l’astérisque :

SELECT * 
FROM employees;

Remarque : On utilise SELECT * pour des tests rapides, mais il est recommandé de nommer les colonnes pour de meilleures performances et clarté.

Résumé

  • Utilisez SELECT FROM pour extraire des données d’une table.
  • Utilisez SELECT * pour récupérer toutes les colonnes d’un coup.
  • Privilégiez la sélection de colonnes nommées dans les applications professionnelles.
